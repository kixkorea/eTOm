1))-"데이터 요금 없는 블루투스 인터폰" 개발을 위한 핵심 로직을 총정리해 드립니다.
이 앱은 '전화'가 아니라 '실시간 오디오 데이터 스트리밍' 앱입니다.
* 통신 방식: Bluetooth RFCOMM (Serial Port Profile). 가장 안정적으로 연속 데이터를 보낼 수 있습니다.
* 전송 모드: Full-Duplex (전이중 통신). 양방향 마이크와 스피커를 동시에 활성화합니다.
* 데이터 흐름: * 내 마이크 → 바이트(Byte) 배열 변환 → 블루투스 소켓(Socket) 전송
* 블루투스 소켓 수신 → 바이트 배열 → 상대방 이어폰 출력
2. 필수 기술 스택 (Android 기준)
가장 간편한 구현을 위해 다음 API들을 조합합니다.
| 단계 | 사용 API / 기술 | 역할 |
|---|---|---|
| 연결 | BluetoothServerSocket / BluetoothSocket | 두 폰을 무선으로 묶어주는 통로 |
| 입력 | AudioRecord | 마이크 소리를 실시간 디지털 데이터로 캡처 |
| 출력 | AudioTrack | 받은 디지털 데이터를 즉시 소리로 재생 |
| 처리 | Thread (또는 Coroutines) | 끊김 없는 송수신을 위한 백그라운드 작업 |
3. 개발 단계별 체크리스트
* 권한 설정: BLUETOOTH_CONNECT, RECORD_AUDIO, NEARBY_DEVICES 권한 확보.
* 소켓 연결: 한 기기는 Server(대기), 다른 기기는 Client(연결)가 되어 파이프라인 형성.
* 오디오 루프 (Audio Loop):
* AudioRecord.read()로 소리를 읽어와서 OutputStream.write()로 전송.
* InputStream.read()로 받은 데이터를 AudioTrack.write()로 재생.
* 에코 캔슬링: 각자 이어폰을 끼면 하울링이 거의 없지만, 시스템의 AcousticEchoCanceler를 활성화하면 더 깔끔해집니다.
4. 에이전트에게 입력할 "한 줄 프롬프트"
안티그래비티 에이전트에게 다음과 같이 명령하세요.
> "안드로이드 코틀린으로 Bluetooth RFCOMM을 이용한 실시간 양방향 오디오 스트리밍 앱을 만들어줘. AudioRecord로 입력받아 BluetoothSocket으로 전송하고, 수신된 데이터는 즉시 AudioTrack으로 재생하는 구조여야 해. 무전기 방식이 아닌 항시 연결된 통화 방식이어야 하며, 이어폰 사용을 전제로 해줘."

5. 이 방식의 장점 (냉정한 결론)
* 비용 0원: 통화료, 데이터 요금이 전혀 없습니다.
* 독립성: 기지국이 없는 산속이나 지하에서도 작동합니다.
* 간결함: 서버 구축이 필요 없고 앱 두 개만 있으면 끝납니다.

2))-방을 만들고 연결한다'는 개념은 프로그래밍적으로 서버(Server)와 클라이언트(Client) 모델을 블루투스에 적용하는 과정입니다. 인터넷망 없이 기기끼리 직접 대화하기 위한 '통로'를 여는 작업이라고 이해하시면 됩니다.
상세한 메커니즘을 4단계로 정리해 드립니다.
1. 전제 조건: 고유 식별자 (UUID)
두 기기가 서로를 알아보기 위한 '전용 채널 번호' 같은 것이 필요합니다. 이를 UUID라고 합니다. 앱 코드 내에 똑같은 UUID를 심어놓아야 다른 블루투스 기기와 섞이지 않고 두 기기만 연결됩니다.
2. 기기 A: "방 만들기" (Server 모드)
방을 만드는 스마트폰은 상대방이 접속할 때까지 기다리는 서버 역할을 합니다.
* 권한 확인: 블루투스가 켜져 있는지, 주변 기기 검색 권한이 있는지 확인합니다.
* 대기 상태 진입: listenUsingRfcommWithServiceRecord(이름, UUID) 함수를 호출합니다.
* 수락 대기: accept()라는 명령어를 실행하고 대기합니다. 이때 앱은 상대방이 "노크" 할 때까지 멈춰 있는 상태가 됩니다.
* 연결 완료: 상대방이 접속하면 소켓(Socket, 통신 통로)이 생성됩니다.
3. 기기 B: "방 참여하기" (Client 모드)
방을 찾는 스마트폰은 서버를 찾아가서 연결을 요청하는 클라이언트 역할을 합니다.
* 기기 검색: 주변의 블루투스 기기를 스캔합니다.
* 대상 선택: 목록에서 상대방의 기기(방 만든 폰)를 선택합니다.
* 연결 요청: 상대방의 UUID를 향해 createRfcommSocketToServiceRecord(UUID)를 호출한 뒤, connect()를 실행합니다.
* 성공: 서버가 이를 수락하면 통로가 열립니다.
4. 연결 후: 통화 시작 (Data Streaming)
연결(Socket)이 성공하면, 이제 실시간 빨대가 꽂혔다고 생각하시면 됩니다.
* 입력 루프: 내 폰 마이크로 들어온 소리(AudioRecord)를 1초에 수십 번씩 쪼개서 소켓의 OutputStream에 던집니다.
* 출력 루프: 소켓의 InputStream으로 들어오는 데이터를 실시간으로 받아서 스피커(AudioTrack)로 즉시 재생합니다.
* 무한 반복: 이 작업을 앱을 끌 때까지 별도의 백그라운드 스레드(Thread)에서 무한히 반복하면, 지연 시간 없는 실시간 통화가 구현됩니다.
